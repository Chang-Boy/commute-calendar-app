## 개요
PRD 또는 SPEC 문서를 분석하여 `docs/PLAN.md` 형태의 단계별 구현 계획서를 작성하는 스킬.
기존에 작성된 PLAN.md가 존재한다면 overwrite를 진행한다.

## 사용 방법
```
구현 계획서 작성해줘
```

## 실행 절차

### 1단계: 사전 검증
- [ ] `docs/PRD.md` 또는 `docs/SPEC.md` 파일 존재 확인
- [ ] PLAN.md가 이미 존재하면 사용자에게 기존 파일 존재 여부 확인 후 진행
- [ ] 문서에 모호하거나 부족한 부분이 있으면 **반드시 사용자와 논의 후 결정**

### 2단계: 계획 작성
- [ ] 전체 프로젝트의 작업 내용 요약
- [ ] CLAUDE.md의 코딩/디자인 컨벤션 검토
- [ ] 적절한 구현 단위로 여러 Phase로 분류
  - 각 Phase는 1-2개의 명확한 단위 작업
  - 데이터 레이어 → 화면 구성 → 통합 순서 권장
- [ ] 각 Phase는 CLAUDE.md의 아키텍처 규칙 철저히 준수
- [ ] 각 Phase마다 체크 가능한 체크박스 리스트 작성

### 3단계: 결과 저장
- [ ] 계획서를 `docs/PLAN.md`로 저장
- [ ] 사용자에게 계획 내용 요약 제시

## 계획서 작성 규칙
- PLAN.md는 Phase 기반의 구조화된 마크다운
- 각 Phase는 다음 항목 포함:
  - Phase 제목 및 개요
  - 핵심 작업 목록
  - 파일 구조/수정 대상
  - Phase 완료 체크리스트
- Phase 간 의존성 명확히 표시
