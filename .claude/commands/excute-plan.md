## 규칙
- 각 Phase는 반드시 **서브에이전트**에서 실행한다
- 진행 도중 이상한 부분이 있으면 사용자와 논의한다 (단, 사용자가 알아서 진행하라고 한 경우는 그냥 작업을 진행한다)

## 개요
`docs/PLAN.md`에 정의된 구현 계획을 단계별로 실행하는 명령어
작업 완료 후 검증 및 체크리스트 관리를 자동으로 수행

## 사용 방법
```
계획 실행해줘
다음 phase 진행
phase 1 실행
```

## 실행 절차

### 1단계: 계획 확인
- [ ] `docs/PLAN.md` 파일 존재 여부 확인
- [ ] 현재 진행 상황(완료된 Phase) 파악
- [ ] 사용자에게 진행 상황 보고

### 2단계: Phase 작업 수행
- [ ] 현재 Phase의 모든 작업 항목 실행
- [ ] CLAUDE.md의 절대규칙 준수:
  - 적절한 규모의 커밋
  - 정적 검사 실행 (flutter analyze, dart fix)
  - 불필요한 주석 삭제 금지
  - 사용자의 명시적 지시 없이 commit/merge 금지

### 3단계: Phase 완료 검증
- [ ] 정적 분석 실행 (`flutter analyze`)
- [ ] 자동 수정 적용 (`dart fix --apply`)
- [ ] 버그/문제 탐색 및 해결
- [ ] 문제 없을 시 PLAN.md의 Phase 체크박스 체크
- [ ] 작업 내용 사용자에게 보고

### 4단계: 다음 Phase 진행
- [ ] Phase 완료 후 사용자의 명시적 지시 대기
- [ ] 사용자 지시 시 다음 Phase 진행 (2-4단계 반복)

### 마지막 단계: 작업 완료
- [ ] 모든 Phase 완료 확인
- [ ] 전체 작업 내용 요약
- [ ] 최종 정적 검사 실행
- [ ] 완료 보고

## 주의사항
- **Phase는 순서대로 진행**: 이전 Phase 완료 없이 다음 Phase 진행 금지
- **사용자의 명시적 지시만 따르기**: "다음 phase 진행" 같은 명확한 지시 필요
- **검증 단계 필수**: lint/analyze 없이 다음 Phase 진행 금지
- **CLAUDE.md 준수**: 모든 작업은 프로젝트 규칙 준수
