# 근태 입력 방식 변경

## 개요
현재는 사용자가 필수적으로 출퇴근 시간을 cupertinoPicker를 활용해서 입력해야한다.
하지만 이 앱을 사용하는 사용자들은 출퇴근 시간도 중요하지만, 전체 출근시간(9h 10m 과 같은)이 더 중요하다
따라서 FAB를 눌러 근태정보를 입력받을 때, 가장 기본적으로 몇시간 몇분 근무했는지를 Textfield로 입력받는다.
그리고 아래에 선택적으로 "출퇴근 시간 입력" 버튼을 눌러 cupertinoPicker로 nullable로 입력받도록 개선한다.

## 모델링
- 기존에 구현된 entity와 model에서, 필요한 nullable 처리를 적용한다.

## supabase 세팅
- 기존 테이블들도 nullable 등을 처리한다.

## Data 레이어 작성
- CalendarDataSource 관련 수정사항들을 적용한다

## 도메인 레이어 작성
- 도메인 레이어에서 수정사항이 있다면 적용한다.

## UI 레이어 작성 
- Bloc에서 수정사항이 있다면 적용한다.
- View쪽에서는 다음과 같은 플로우를 가진다
1. FAB -> 근무 선택
2. 근무시간 필수 입력
3. 바로 하단의 버튼을 누르면 dialog가 뜨며 내부에는 cupertinoPicker로 출퇴근시간 입력받기 (더 좋은 UI 플로우가 있다면 말해줘)
4. 메모 등은 전과 동일
- 출퇴근시간, 총 근무시간 등과 관련된 UI들 중, 필수로 수정해야할 UI가 있다면 적절한 디자인으로 수정을 진행한다.

## 추가 사항
- 엣지케이스와 에러케이스를 잘 잡을 수 있도록 한다.

## 참고사항
- 모든 구현은 단순하게, 복잡하지 않고 가독성 좋게 구현되어야한다.
- 추상체는 Repo만 만든다.
- 기존 디자인 시스템과 테마를 반드시 준수할 것
