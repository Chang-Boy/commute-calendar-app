# 근태 관리 달력 (Commute Calendar) - PRD

## 1. 프로젝트 개요

### 목표
자율 출퇴근제 환경에서 월 필요 출근 시간을 관리하고, 매일의 출퇴근 기록을 통해 잔여 출근 시간을 자동 계산하는 앱

### 기본 개념
- **월 필요 출근 시간**: (월 근무일 × 9시간) - 공휴일 - 휴가
- **자율 출퇴근제**: 하루 9시간 고정이 아니라, 한 달 총 필요 시간만 채우면 됨
  - 예: 월요일 4시간, 화요일 15시간 등 일정이 유연함
- **잔여 출근 시간**: 월 필요 시간 - 실제 기록 시간

### 사용 대상
- 1차: 본인(개인용)
- 2차: 추후 팀원들과 공유 가능하도록 설계

## 2. 핵심 기능

### 2.1 출퇴근 시간 기록
- **수동 입력**: 사용자가 매일 출퇴근 시간을 직접 입력
- **시간 저장**: 날짜별로 출근 시간 저장
- **수정 가능**: 과거 기록 수정 가능

### 2.2 월별 필요 출근 시간 자동 계산
- **공휴일 제외**: 
  - holidays 패키지로 한국 공휴일 자동 인식 (설날, 추석, 정월대보름 등 음력 기반 포함)
  - 국경일, 경축일 자동 제외
- **휴가 제외**: 
  - 사용자가 수동으로 표시 (연차, 반차, 반반차)
  - 연차: 1일 = 9시간
  - 반차: 0.5일 = 4.5시간
  - 반반차: 0.25일 = 2.25시간
- **자동 계산**: 
  ```
  월 필요 출근 시간 = (월 전체 근무일 - 공휴일 - 휴가) × 9시간
  ```

### 2.3 잔여 출근 시간 표시
- **실시간 계산**: 
  ```
  잔여 출근 시간 = 월 필요 출근 시간 - 실제 기록 시간
  ```
- **시각적 피드백**: 
  - 진행률 표시 (진행중, 목표 달성, 초과)
  - 잔여 시간이 음수면 시각적으로 구분

### 2.4 캘린더 뷰
- **table_calendar 패키지 사용**
- **월별 표시**:
  - 공휴일: 특정 색상/스타일로 표시
  - 휴가 표시 (연차, 반차, 반반차 구분)
  - 실제 기록한 출근 시간 표시
  - 오늘 날짜 강조
- **상호작용**:
  - 날짜 선택 시 상세 정보 표시
  - 해당 날짜의 출퇴근 시간 입력/수정

## 3. 사용자 시나리오

### 시나리오 1: 월 초 현황 파악
1. 앱 열기 → 현재 월의 달력 표시
2. 월 필요 출근 시간 확인 (자동 계산됨)
3. 공휴일, 휴가 확인

### 시나리오 2: 출퇴근 시간 기록
1. 오늘 날짜 선택
2. 출근 시간 입력 (또는 수정)
3. 저장 → 잔여 시간 자동 업데이트

### 시나리오 3: 휴가 표시
1. 달력에서 휴가 예정일 선택
2. 휴가 유형 선택 (연차/반차/반반차)
3. 저장 → 월 필요 출근 시간 자동 재계산

### 시나리오 4: 월별 통계 조회
1. 월 선택
2. 해당 월의:
   - 월 필요 출근 시간
   - 실제 기록 시간
   - 잔여 출근 시간
   - 진행률 확인

## 4. 데이터 모델

### 4.1 주요 엔티티

#### `attendance` 테이블
```
- id: UUID (Primary Key)
- user_id: UUID (Foreign Key)
- date: DATE (고유: user_id + date)
- hours: DECIMAL (기록한 시간)
- created_at: TIMESTAMP
- updated_at: TIMESTAMP
```

#### `vacation` 테이블
```
- id: UUID (Primary Key)
- user_id: UUID (Foreign Key)
- date: DATE (고유: user_id + date)
- type: ENUM (연차, 반차, 반반차)
- created_at: TIMESTAMP
- updated_at: TIMESTAMP
```

#### `users` 테이블 (미래 확장용)
```
- id: UUID (Primary Key)
- email: STRING
- name: STRING
- department: STRING (선택사항)
- created_at: TIMESTAMP
```

### 4.2 계산 로직 (앱에서 수행)
- 공휴일: holidays 패키지로 실시간 계산
- 월 필요 시간: 달력 일 수 - 공휴일 - 휴가(시간 환산) = 근무일 × 9시간
- 잔여 시간: 월 필요 시간 - (DB의 attendance 합계)

## 5. 화면 구성

### 5.1 메인 화면 (홈)
- **상단**: 
  - 현재 월 선택/변경
  - 월 필요 출근 시간, 실제 기록 시간, 잔여 시간 요약
  - 진행률 바
- **중단**: 
  - table_calendar (월별 캘린더)
  - 공휴일, 휴가, 기록 시간 시각화
- **하단**: 
  - 오늘 또는 선택한 날짜의 상세 정보
  - 출퇴근 시간 입력/수정 버튼

### 5.2 상세 입력 화면
- **날짜**: 선택한 날짜 표시
- **출근 시간**: 텍스트 입력 (숫자만)
- **휴가 설정**: 토글 또는 드롭다운 (연차/반차/반반차)
- **저장 버튼**: 데이터 저장 및 월 통계 자동 업데이트

### 5.3 월별 통계 화면 (미래 기능)
- 월 선택
- 요약 통계: 필요 시간, 기록 시간, 잔여 시간
- 주별/일별 상세 기록 (테이블 또는 차트)

## 6. 기술 스택

| 계층 | 기술 | 선택 사유 |
|------|------|---------|
| **Frontend** | Flutter | iOS, Android, macOS, Windows 동시 지원 |
| **Backend** | Supabase | PostgreSQL 기반, 실시간 동기화, 인증 제공 |
| **캘린더 UI** | table_calendar | 사용자 정의 가능, 한국어 지원 |
| **공휴일** | holidays 패키지 | 한국 공휴일 자동 처리 (음력 포함) |
| **상태 관리** | flutter_bloc | BLoC 기반 상태 관리 |
| **의존성 주입(DI)** | GetIt | 싱글톤 및 서비스 로케이터 패턴 |
| **인증** | Supabase Auth | (추후: 개인 → 팀 확대 시) |
